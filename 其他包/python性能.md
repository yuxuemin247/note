1、python(`CPython`)性能慢的常见原因：

- GIL(全局解释器锁),对GIL的需求主要时由`Cpython`的内存管理算法导致的

  - 如果应用程序时单线程、单解释器的，那么不会对速度有任何影响。去掉GIL不会影响代码的性能

- 解释性语言不是编译语言

  ```
  当 python test.py 执行时，Cpython会启动一长串操作，包括读取、语法分析、编译、解释和执行。
  在编译阶段会生成.pyc文件字节码，字节码会写搭配_pycache_下的文件。
  因此绝大数情况下(除非第一次运行)，python时在解释字节码并在本地执行。
  java将源代码编译成中间语言，然后java虚拟机读取字节码并即时编译成机器码。
  即使编译(jit) JIt本身并不能执行更快，但是好的Git优化器能找到应用程序执行最多的部分，称为"热点"，然后对字节码进行优化，将它们替换成效率更高的代码。
  ```

  - Git本身启动速度就慢，`Cpython`启动速度也很慢,所以`Cpython`不用`Jit`

- 动态类型语言

  - python 可以在运行时动态替换对象的方法，可以在运行时给底层系统调用打补丁。
  - 静态语言要求在变量定义时指定其类型，例如 C、 C++、 Java,而动态类型尽管有类型的概念，但变量的类型是动态的。
  - 不需要定义类型并不是python慢的原因，但这种设计使得python的优化变得很困难。
  - 动态类型每次读取、写入、或引用都要检查类型

```
python 慢的主要原因时因为它的动态性。它用于解决各种问题。
通用就cpython
对于启动时间不重要，可以利用jit,可以考虑使用PyPy
对于代码中性能很重要的部分，如果变量大多时静态类型，有Cython,它通过结合c的静态类型和python的方式，使得代码中的类型已知，从而优代码。
```
