1. 原型设计、概要设计阶段：

   1. 产品人员进行原型设计, 提出详细的开发需求，大概率这个时候，是去抄一个同业产品

      1. 用户模块：
         1. 获取验证码
         2. 验证接口
         3. 获取用户资料
         4. 修改用户资料
         5. 上传头像
      2. 社交模块：
         1. 获取推荐用户
         2. 喜欢、超级、不喜欢
         3. 反悔
         4. 查看喜欢我的人
         5. 最受欢迎的人：排行榜

   2. 需求文档：产品出需求文档(用户使用的流程、高保真产品原型，各种异常分支如何处理、如何提醒用户，凡是用户看得到的东西，需求文档里要说明白)，详细需求讨论会，产品出原型图![149724028970083396](/Users/zebin/Downloads/149724028970083396.jpg)

   3. 概要设计：技术出概要设计

      1. 数据库：MySQL，没特殊的 5.7
      2. 框架：Django，2.2
      3. 服务器：Ubuntu 1804 LTS

   4. 接口文档：

      1. 用户模块：

         1. 获取验证码

            1. 描述：

            2. 方法：POST

            3. url：/user/vcode

            4. 请求：

               ```http
               phone=185xxx
               ```

               

            5. 响应：

               1. ```json
                  // 成功的
                  {
                    "code": 0,
                    "msg": "",
                    "data": null
                  }
                  // 失败的
                  {
                    "code": 500,
                    "msg": "inner error",
                    "data": null
                  }
                  ```

                  

         2. 验证接口

            1. 描述：

            2. 方法：POST

            3. url：/user/verify

            4. 请求：

               ```http
               phone=185xxx&vcode=3456
               ```

            5. 响应：

               ```json
               // 成功的
               {
                 "code": 0,
                 "msg": "",
                 "data": {
                   id:112,
                   nickname:"zhaoyun",
                   age:34,
                   ...
                 }
               }
               // 失败的
               {
                 "code": 5000,
                 "msg": "vcode error",
                 "data": null
               }
               ```
